# afo-strategy Agent 指南

## 模块定位

`afo-strategy` 承载企业级策略管理能力。当前包含模型准入管理，后续可放置与模型调用门禁、路由策略、企业级策略基线相关的控制面代码。

## 边界

- 模型准入管理只负责企业/租户允许或禁止调用哪些模型，以及未配置模型的默认准入模式。
- 不在本模块实现 API Key 生命周期、Key 级授权、模型参数策略、预算额度、计费或智能路由。
- 控制面接口可兼容历史 `gateway:whitelist:*` 权限编码，但代码命名应使用 `ModelAccessPolicy` 语义。

## 数据库

- 新增表结构 SQL 放在 `src/main/resources/sql/postgres/`。
- SQL 固定按 PostgreSQL 编写，业务表必须使用 `afo_` 前缀并包含租户、审计和逻辑删除字段。
- 表结构变更不由代码自动执行，交付时说明 SQL 文件路径，由使用者手动执行。

## 代码风格

- Controller、Service、Mapper、Domain、VO 保持职责分离。
- 保存模型准入策略时必须校验模型目录存在性、允许/禁止列表互斥、生效时间合法性，以及启用策略时间区间不重叠。
- 没有活跃策略时必须按兼容策略放行，避免上线后误拒绝历史请求。
